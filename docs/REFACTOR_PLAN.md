# Vim 配置重构计划

## 🎯 目标：简化结构，保持功能

### 当前问题
- 6个脚本功能重叠（deploy.sh, setup_vim.sh, install_plugins.sh 等）
- 配置文件分散（.vimrc + macvim_config.vim）
- 文档冗余（3个教程文档）
- 维护复杂度高

### 重构原则（奥卡姆剃刀）
1. **一个功能，一个文件**
2. **合并重复逻辑**
3. **统一入口点**
4. **保持所有现有功能**

## 📁 重构后的目录结构

```
vim_settings/
├── vimrc                    # 统一配置文件（合并.vimrc + macvim_config.vim）
├── vim-manager              # 统一管理脚本（合并所有.sh）
├── README.md                # 精简使用说明
├── plugins/                 # 插件目录（不变）
├── colors/                  # 主题目录（不变）
└── docs/                    # 文档目录
    └── vim-guide.md         # 完整的 Vim 指南
```

## 🔧 重构步骤

### 1. 配置文件整合
**目标**: 将 .vimrc (668行) + macvim_config.vim (236行) 合并为一个智能配置文件

**优势**:
- 单一配置文件，易于维护
- 自动检测环境（terminal/GUI）
- 条件加载配置块

### 2. 脚本整合
**目标**: 将6个脚本合并为1个多功能脚本

```bash
./vim-manager install        # 安装配置和插件
./vim-manager update         # 更新插件
./vim-manager remove [name]  # 删除插件
./vim-manager config         # 配置管理
./vim-manager start          # 启动 MacVim
./vim-manager help           # 帮助信息
```

### 3. 文档整合
**目标**: 精简文档结构

- `docs/vim-guide.md` - 传统 Vim 完整指南

### 4. 智能配置检测
**新增功能**:
- 自动检测操作系统
- 自动检测 Vim 类型（vim/nvim/macvim）
- 自动检测终端能力
- 按需加载配置块
