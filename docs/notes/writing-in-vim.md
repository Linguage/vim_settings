# 在vim上的写作体验

目前使用vim进行写作,体验不敢说特别好,但也还OK。

虽然在并不是很看得上那些一个破烂工具使用一万年的老登，但是当前不缺写作的工具，试来试去反倒显得专注成了特别稀缺的状态。

我写这个文档的主要目的还是去习惯vim的一些操作，不为了写什么有价值的内容，而是为了勾勒自己的心路历程。


## 我使用vim做什么？

我不是一个程序员。一款编辑器在上手阶段，我喜欢尝试的通常都是它的写作功能，尤其是md格式的写作。之前使用vs code，我尝试了其md插件得有五六款，以至于后来我自己都分不清楚哪一款是什么样。后来上手Emacs，我体验了org mode，持续了几个月，后来也不怎么用了，因为我发现现在的笔记软件的丰富度普遍很高，很多功能虽然也可以使用Emacs进行配置，但我已经失去了捣鼓这些工具的欲望。



1. 预览vim中的md文档。
2. 写点代码？
3. 写一个数学文档,里面要有很多数学公式


使用的vim的挑战在于，自己被全新的一套规矩框住，而且这套规矩还很硬，根本不容商量，就像来到了一个非常陌生的地方一样，除了肢体语言，几乎没有任何其它的方式与别人交流。

我之前使用过Emacs，并在Emacs中启用过Evil模式。这种模式可以使用Vim的快捷键，但总是有退路，一旦不开心，接直接把该插件停用，恢复到普通键位的操作。类似的情况也发生在其它的环境中，比如在vs code和Pycharm等环境中，我也有vim键位插件，但都半途而废。


这一次，真的到了要把vim纳入自己的工具版图中的时候了。我开始使用注入warp之类的命令行工具，而且在使用mac的过程中也经常感受到熟悉vim命令的必要性。

有一种感觉：使用nvim的体验也不错。不过我的nvim直接套用的lazyvim中的配置，内容过于丰富，我反倒无所适从。

但是我依然计划在后续逐渐切换到nvim，而当前的vim配置主要还是进行学习与测试，通过这个过程逐渐书写vim的基本操作。

我有时候喜欢捣鼓一些工具，但重造轮子还是需要耗费不少心血，我不想做这些事情。

我现在处于一个特殊的节点，因为我不知道自己需要做什么。我对未来缺乏规划，所以难免陷入东一榔头西一棒子的状态。我确实可以做出一些东西，只不过如果没有聚焦，我不可能做出很成规模的产品。
